# Auto-posting blog posts to LinkedIn and X/Twitter

The announcement text for each network is **committed in the blog post's frontmatter**
and reviewed in the same PR as the post - it is not a template. Posts publish to
social at Buffer's scheduled slots (14:00-15:00 European time, the best window for
worldwide impressions).

## Writing a post that gets shared

Add a `social:` block to the post's frontmatter. The copy is the exact text that
will be posted; the canonical post URL is appended automatically on its own line.

```yaml
---
title: "My new post"
date: 2026-08-01
categories: ["FastAPI"]
social:
  linkedin: |
    The LinkedIn announcement, written for a LinkedIn audience.

    Multiple paragraphs are fine.
  twitter: "The X announcement - max 250 chars, the link takes the rest."
---
```

- **No `social:` block → the post is not shared anywhere.** Sharing is opt-in, per post,
  per network (you can provide only one of the two).
- The build fails if the twitter copy exceeds 250 chars (X counts links as 23).

## How it flows

```
merge post with social copy -> site deploys
  -> /rss/linkedin.xml and /rss/twitter.xml gain an item whose body IS the copy
  -> IFTTT detects the new item and adds it to the matching Buffer channel queue
  -> Buffer publishes at the next scheduled slot (14:00/15:00 CET)
```

Posts wait in the Buffer queue until a slot, so merge timing doesn't matter. For a
one-off exact time (e.g. 16:00), open the queued post in Buffer and schedule it
directly - a custom time overrides the slots.

## One-time setup

### 1. Buffer ([buffer.com](https://buffer.com))

1. Connect two channels: your LinkedIn profile and X account (free plan covers this).
2. For each channel, set Settings → Posting Schedule: timezone `Amsterdam (Europe)`,
   daily slots at `14:00` and `15:00`.

### 2. IFTTT ([ifttt.com](https://ifttt.com))

Buffer has no native RSS ingestion, so two IFTTT applets bridge feed → queue:

| Applet | If (RSS feed) | Then (Buffer channel) | Body |
| --- | --- | --- | --- |
| LinkedIn | `https://marcelotryle.com/rss/linkedin.xml` | LinkedIn | `{{EntryContent}}` |
| X | `https://marcelotryle.com/rss/twitter.xml` | X | `{{EntryContent}}` |

The body is just `{{EntryContent}}` - the committed copy plus the post link, nothing
templated. When you first enable an applet, IFTTT treats existing feed items as seen
and only fires for items added afterwards; watch the first real post go through to
confirm.

## Feeds

- `/rss.xml` - the human/reader feed: every post, with title, excerpt, and link.
- `/rss/linkedin.xml` and `/rss/twitter.xml` - automation feeds generated by
  `src/lib/social-feed.js`: only posts with committed copy for that network, and the
  item body is the copy itself with the canonical URL appended.
